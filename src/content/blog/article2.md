## Capture The Decision

Useful technical writing does not need to be long. Start with the decision you made, the constraints you had, and the trade-offs you accepted.

## Keep Examples Small

Readers usually need a concrete example, not a full production system. A short code snippet or diagram can make the article easier to remember.

## Publish Iteratively

Write the first version for clarity. You can always add more detail when readers ask better questions.
